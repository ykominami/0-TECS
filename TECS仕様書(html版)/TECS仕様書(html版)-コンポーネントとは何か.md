# コンポーネントとは何か

TECSにおける**コンポーネント**とは、独立して再利用可能なソフトウェアの構成単位です。TECSコンポーネントモデルでは、コンポーネントは以下の2つの概念に分かれます。

| 概念 | 説明 |
|------|------|
| **セルタイプ (Celltype)** | コンポーネントの**型**定義。属性・受け口・呼び口を宣言する。 |
| **セル (Cell)** | コンポーネントの**インスタンス**。実行時の実体。 |

## コンポーネントの構成要素

| 要素 | 説明 |
|------|------|
| **セル (Cell)** | コンポーネントのインスタンス。実行時の実体。 |
| **セルタイプ (Celltype)** | コンポーネントの型定義。属性・受け口・呼び口を宣言。 |
| **シグニチャ (Signature)** | コンポーネント間インタフェースの関数頭部集合。 |
| **受け口 (Entry)** | シグニチャを実装して外部に提供するポート（提供側）。 |
| **呼び口 (Call)** | シグニチャに対応する外部サービスを利用するポート（利用側）。 |
| **属性 (Attribute)** | セルの静的パラメータ（設定時に値が決まる）。 |
| **変数 (Variable)** | セルの動的状態（実行時に変化する内部状態）。 |

**結合**: 呼び口と受け口を接続することでコンポーネント間の依存関係を宣言します。一つの受け口に複数の呼び口を結合できますが、一つの呼び口が接続できる受け口は一つです。
