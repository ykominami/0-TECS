# TECSコンポーネント図の書き方を教えて

TECSコンポーネント図（TCD）で使用できるすべての表記パターンの概要です。

## 基本図

```
+------------------+      sSignature      +------------------+
| tCellA           |                      | tCellB           |
|    cCall ------->|--------------------->|> eEntry          |
+------------------+                      +------------------+
```

## 合流（多対一結合）

一つの受け口に複数の呼び口が結合できます。合流点では3本の線分が集まります。

```
| CellA.cCall ------\
|                    >----> |> eEntry  (tCellB)
| CellB.cCall ------/
```

## 呼び口配列

```
| cArr[0] ---->  |> ePort0 (Cell0)
| cArr[1] ---->  |> ePort1 (Cell1)
```

## 受け口配列

```
+------------------+
| tCellB           |
|   eArr[0] <|     |
|   eArr[1] <|     |
+------------------+
```

## コールバック（逆結合）

`[callback]` シグニチャを使った逆結合（`<=`）。受け口から呼び口へ向かう矢印になります。

```
+------------------+   sCallback   +------------------+
| tCaller          |               | tService         |
|         cCb <====|===============|= eCbEntry        |
+------------------+               +------------------+
```

## アクティブ（二重線）

`[active]` セルタイプのセルは長方形の辺を**二重線**で描きます。

```
+==================+
|| tMyTask         ||
||                 ||----> cSysLog
+==================+
```

## 複合セル

外側の長方形が複合セルを示します。内部セルを囲みます。

```
+------------------------------------------+
| tMyComposite (composite)                 |
|   +----------+         +----------+      |
|   | tInner   |-------->| tInner2  |      |
|   +----------+         +----------+      |
+------------------------------------------+
```

## RPC（スループラグイン）

リージョン間のスループラグインを経由する結合（`[through(Plugin)]`）。

```
[rRegion1]           [through(RPCPlugin)]          [rRegion2]
| CellA.cPort |----//========================//---->|> ePort (CellB)
```

## アロケータ

`[allocator()]` 指定のある呼び口は特別な表記で示します。

## 多段リレーモデル

複数のリレーアロケータを経由するモデルの表記。メモリバッファが複数のコンポーネントを経由する場合に使用します。
