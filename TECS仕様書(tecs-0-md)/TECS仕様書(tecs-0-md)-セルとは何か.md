# TECS仕様書(tecs-0-md)-セルとは何か

セルは、TECSにおける**ソフトウェア部品のインスタンス（コンポーネントインスタンス）**である。セルはいずれかのセルタイプまたは複合セルタイプに属し、セル名を持つ。セルは静的に生成されコンパイル時に実行モジュールに格納される。

---

## 構成要素

セルは以下の要素を持つことができる：

| 要素 | 説明 |
|------|------|
| **受け口** (entry port) | セルが機能を提供するための口。シグニチャに対応付けられる |
| **呼び口** (call port) | 他のセルの機能を利用するための口。シグニチャに対応付けられる |
| **属性** (attr) | セル内部の定数値。生成後変更不可。ROM化システムではROMに配置 |
| **内部変数** (var) | 動的に変更可能なRAM変数 |

## 属性と内部変数の違い

| | 属性 (attr) | 内部変数 (var) |
|-|-------------|---------------|
| 変更 | 不可（定数） | 可能（動的） |
| 配置 | ROM（ROM化時） | RAM |
| 初期値個別指定 | 可 | 不可（属性を介して間接的には可） |

属性が個別に初期値を持てるのに対し、内部変数の初期値は個別指定不可。これはROM化システムにおいてRAM初期値保持のROM領域消費を避けるための仕様決定。

## ID と IDX

- **ID**: セルに割付けられる整数値
- **IDX**: セルを指すポインタ情報またはID。通常はポインタが用いられる

## アクティブ属性

セルが**アクティブ**とは、タスク・ハンドラ等CPU資源割付けを伴うセル、またはそれらを内包するセルのこと。アクティブかどうかはセルタイプで指定される。

- アクティブセルは受け口を一つも持たなくてもよい
- 非アクティブで受け口も未結合かつファクトリを持たないセルは誤り（呼び出されない）

## シングルトン

セルタイプに**シングルトン**を指定すると、そのセルタイプに属するセルは1つだけに制限される。
