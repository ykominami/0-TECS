# TECS仕様書(tecs-0-md)-コールバックとは何か

TECSにおけるコールバックは、**２つのセルが、それぞれ受け口と呼び口を持ち、相互に接続し合っている状態**における補助的な結合のことである（主体的な結合の呼び先から呼び元への呼び出し）。

---

## 定義（§2.6.1）

コールバックは「主体的な結合において呼び先から呼び元への呼び出しが行われる結合」を指す。TECSの仕様としては、**コールバックをサポートする専用の機構を持たない**。コールバックは主体的な結合とは別の明示的な結合として記述する。

## TECSがコールバック専用機構を持たない理由

1. 呼び口と受け口を結合するだけでコールバックの結合が生じるようにすると、合流してくる場合にどこにコールバックすればよいかの決定ができない
2. コールバックがしばしば主体的な結合とは異なるコンテキストで呼び出される（例：デバイスを操作するセルのコールバックは割込みルーチンから呼び出される）
3. コールバックの結合はITRONのようなOSにおいても動的に行うが、現バージョンのTECSでは静的な結合のみが可能
4. 分散する場合と、しない場合でコンポーネントが同じように動作することを期待した設計方針
