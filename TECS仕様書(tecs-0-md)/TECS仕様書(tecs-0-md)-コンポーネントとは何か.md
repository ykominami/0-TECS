# TECS仕様書(tecs-0-md)-コンポーネントとは何か

TECSにおけるコンポーネントは、**セルタイプ（ソフトウェア部品の型式）** とそのインスタンスである **セル（コンポーネントインスタンス）** の組み合わせによって構成される。

---

## コンポーネントの基本要素

TECSコンポーネントモデルの基本要素：

| 要素 | 英語 | 概要 |
|------|------|------|
| **セル** | cell | コンポーネントインスタンス。受け口・呼び口・属性・内部変数を持つ |
| **シグニチャ** | signature | 関数ヘッダの集合。セル間インタフェースを規定 |
| **セルタイプ** | celltype | ソフトウェア部品の型式。セルの属性・口・動作を定義 |
| **受け口** | entry port | セルが機能を提供する口 |
| **呼び口** | call port | セルが他セルの機能を利用する口 |
| **結合** | join | 呼び口と受け口をつなぐ。呼び元セルで指定 |
| **属性** (attr) | attribute | ROM化可能な定数。セル生成後変更不可 |
| **内部変数** (var) | variable | 動的に変更可能なRAM変数 |

## 静的生成と静的結合

TECSは**静的**を基本とする：
- セルはコンパイル時に生成され実行モジュールに格納
- 結合はコンパイル時に確定し、実行モジュールのデータとして格納
- 現バージョンでは動的生成は未対応（将来の拡張予定）

## コンポーネントを用いた開発手順

1. TECSコンポーネント図でソフトウェア構造を表現
2. シグニチャを記述
3. セルタイプを記述
4. 組上げ記述（セルの配置と結合）
5. TECS CDLをTECSジェネレータ（tecsgen）に通す
6. 生成されたインタフェースコード・テンプレートコードとセルタイプコードをコンパイル・リンク
