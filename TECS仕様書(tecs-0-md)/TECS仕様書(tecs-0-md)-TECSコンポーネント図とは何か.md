# TECS仕様書(tecs-0-md)-TECSコンポーネント図とは何か

TECSコンポーネント図は、**コンポーネントインスタンス（セル）とそれらを組み合わせて構築されるシステムの構造を表現するもの**である（§3.1）。コンポーネントの組上げ（ソフトウェア構造）の直観的な理解を助けるために使用する。

---

## 位置づけ（§3.1）

- 実装されたソフトウェアの構造と一対一に対応する
- 具体的な実装を表現するものであり、ハードウェアの回路図に近い
- TECSコンポーネント間の関係は呼び口と受け口の結合のみ
- TECSのコンポーネントは基本的に静的であり、動的なコンポーネントシステムとは異なり結合が曖昧化されることがない

## 仕様の4要素の一つ

TECS仕様は以下4要素で構成されており、コンポーネント図はその一つ：
- TECSコンポーネントモデル
- **TECSコンポーネント図** — コンポーネントモデルを図により表現
- TECS CDL
- TECSコンポーネント実装

## 基本的な図の要素

| 要素 | 図での表現 |
|------|----------|
| セル | 長方形（セルタイプ名・セル名を記す） |
| 受け口 | 長方形の辺に沿った塗りつぶし三角形（内向き）＋受け口名 |
| 呼び口 | 長方形の辺から伸びる線分＋呼び口名 |
| 結合 | 呼び口から受け口へ向かう線分（沿ってシグニチャ名を添える） |
| アクティブセル | 長方形の辺を二重線 |
| RPC接続 | 二重線による接続表現 |
