# TECS仕様書(tecs-0-md)-シグニチャとは何か

シグニチャは、**関数ヘッダの集合**によってセル間のインタフェースを規定するものである。呼び口や受け口はシグニチャに対応付けられ、同じシグニチャを持つ呼び口と受け口のみが結合できる。シグニチャはシグニチャ名を持つ。

---

## 役割

シグニチャはTECSにおけるAPIの「型」を定義する：
- C言語の関数インタフェースよりも厳密（引数の入出力方向・配列か非配列かを明示）
- 同じシグニチャを満たせば呼び口と受け口が結合可能（形式的一致）
- 関数呼び出しのパケット化（マーシャラ）や呼び出しトレースの自動生成を可能にする

## 関数の引数方向

シグニチャ内の関数引数には以下の方向を指定する：

| 指定子 | 意味 |
|--------|------|
| `in` | 呼び先への入力（非アロケータメモリ） |
| `out` | 呼び先からの出力（非アロケータメモリ） |
| `inout` | 入出力 |
| `send` | 呼び先への入力（アロケータ確保メモリ。受け取り側が解放） |
| `receive` | 呼び先からの出力（アロケータ確保メモリ。受け取り側が解放） |

`send`/`receive` はアロケータにより確保されたメモリ領域を渡すことを示し、受け取った側が解放義務を負う。

## 一方向関数（oneway）

戻り値を返さず出力引数も持たない関数は `oneway` として指定できる。  
→ RPC（リモート呼び出し）時に**非同期呼び出し**（呼び出し完了前に処理再開）が可能になる。

## データ型の逸脱（deviate）

TECSはポインタの多重度を引数の入出力方向ごとに制限する。この制限を満たさない場合を「逸脱」という。シグニチャで逸脱を明示的に宣言しなければならない。逸脱したシグニチャはプラグインによるセル自動生成が原則不可。

## コンテキスト指定子

シグニチャに `context` 指定子でコンテキストを指定できる（§4.7.3）：
- `"task"` — タスク部（未指定の場合のデフォルト）
- `"non-task"` — 非タスク部
- `"any"` — いずれのコンテキストも可能

この指定はコンポーネント設計者・利用者へのメモであり、tecsgenによる検査は行われない。

## アロケータシグニチャ

send/receive引数を持つシグニチャでは、使用するアロケータのシグニチャを定義する必要がある。受け口側のセルタイプで具体的なアロケータを指定する。
