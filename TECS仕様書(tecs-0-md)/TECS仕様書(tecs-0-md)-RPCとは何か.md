# TECS仕様書(tecs-0-md)-RPCとは何か

TECSにおけるRPC（リモートプロシージャコール）は、**通常の呼び出しとは異なり、呼び先の関数が異なるタスクやプロセッサで処理されるリモート呼び出し**である。TECSでは**スループラグイン**によって実現され、通常の結合をRPCチャネルコンポーネントに置き換えることで実現する。

---

## 接続の4分類（§2.14.2）

TECSでは組込みシステムの特性を考慮し、接続を4通りに分類する：

### トランスペアレント接続（アドレス空間共有）

| 種別 | 説明 |
|------|------|
| **直接接続（結合）** | 通常の関数呼び出し。同一プロセスの直接呼び出し |
| **アウトタスク接続** | 同一プロセッサの異なるタスクで呼び先が処理される。メモリ共有前提 |

### オペイク接続（アドレス空間非共有）

| 種別 | 説明 |
|------|------|
| **クローズドネットワーク接続** | 静的に接続可能。システム構成が固定 |
| **オープンネットワーク接続** | 接続先・接続元の存在保証なし。認証等の手続きが必要 |

オペイク接続ではポインタの指す本来の引数をコピーして渡す必要がある。

## RPCの実現（§2.14.3）

RPC = **結合をRPCチャネルコンポーネントに置き換える**

RPCチャネルは以下のコンポーネントから構成される：
- **マーシャラ**: 引数をパケット化（シグニチャごとに専用のものを生成）
- **アンマーシャラ**: パケットを引数に復元
- **チャネル**: 通信路（共有メモリ・CAN・専用RTネットワーク等、交換可能）

マーシャラ・アンマーシャラはシグニチャごとに異なるため、専用プラグイン（tecsgen用）で自動生成する。

## 非同期呼び出し

シグニチャで `oneway` 指定された関数は、RPC時に**非同期呼び出し**が可能。呼び出し完了前に呼び元が処理を再開できる。

## 分散と非分散でのコード一致

「**分散と非分散で同じコード**」を目指す設計方針。スループラグインを挿入する・しないによって分散/非分散を切り替えられ、アプリケーションコード側の変更は最小限になる。
