# TECS仕様書(tecs-0-md)-アロケータとは何か

TECSにおけるアロケータは、**引数の `send`/`receive` 指定を用いてメモリ領域を渡す場合**に使用されるものである。データを渡そうとする側がアロケータでメモリを確保し、受け取った側が解放する規則を定める。セルタイプコード内部のアロケータはTECS仕様の対象外。

---

## 基本規則

- **渡す側**: アロケータでメモリ領域を確保
- **受け取り側**: 受け取ったメモリ領域を解放
- 呼び元・呼び先双方が**同一のアロケータを使用することへの合意**が必要

## シグニチャとの連携

1. **シグニチャ定義時**: `send`/`receive` 引数にアロケータシグニチャを指定
2. **受け口側セルタイプ定義時**: 具体的なアロケータを指定

受け口側で定義する理由：複数の呼び口が合流して1つの受け口に結合される場合、すべての呼び元が同一アロケータを使う必要があるため。

## アロケータの種類

| 種類 | 説明 |
|------|------|
| **通常アロケータ** | メモリ動的割付けの基本形 |
| **アロケータ呼び口** | アロケータセルへの結合口（暗黙的に生成される） |
| **リレーアロケータ** | `send` 指定の引数として受け口に渡されたメモリを、そのまま呼び口から呼び先に引き渡す |
| **デバイスアロケータ** | デバイス固有のメモリ領域に限定したアロケータ（仕様未検討） |

## アロケータ呼び口

呼び口または受け口のシグニチャによってアロケータが指定されている場合、アロケータセルへの結合（アロケータ呼び口）が暗黙的に生成される。
