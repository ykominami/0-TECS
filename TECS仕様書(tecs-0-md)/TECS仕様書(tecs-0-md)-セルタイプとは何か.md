# TECS仕様書(tecs-0-md)-セルタイプとは何か

セルタイプは、**ソフトウェア部品の型式**を定義するものである。セルタイプはセルタイプ名を持ち、セルが持つ各要素（呼び口・受け口・属性・内部変数）のシグニチャや型を定義する。セルの動作を実装するコード（セルタイプコード）はセルタイプごとに記述される。

---

## セルタイプとセルの関係

- **セルタイプ** = ソフトウェア部品の型（クラスに相当）
- **セル** = そのセルタイプのインスタンス（コンポーネントインスタンス）
- 同じセルタイプに属するセルは、同じ口・属性・変数・機能を持つ

## セルタイプが定義するもの

| 要素 | 内容 |
|------|------|
| 受け口 | 名前・対応シグニチャ |
| 呼び口 | 名前・対応シグニチャ・optional指定 |
| 属性 | 名前・型・初期値・指定子（omit等） |
| 内部変数 | 名前・型 |
| セルタイプコード | C言語による振る舞いの実装 |

## セルタイプコード

セルタイプコードはC言語で記述し、以下を含む：
- **受け口関数**: 受け口に対応する関数の実装（セルタイプ独自の処理）
- **呼び口関数呼び出し**: 呼び口を通じた他セルの機能利用
- **CBポインタ (p_cellcb)**: セル管理ブロック（CB）へのポインタ経由で属性・変数にアクセス

ファイル名は `<セルタイプ名>.c` (非シングルトン) または `<セルタイプ名>_s.c` (シングルトン) が慣例。

## 指定子

セルタイプには以下の指定子を付与できる：

| 指定子 | 意味 |
|--------|------|
| `singleton` | セルを1つに制限 |
| `active` | タスク等CPU資源割付けを伴う能動的セル |
| `prototype` | プロトタイプ宣言（実体は別で定義） |

## CDLでの記述例

```
celltype tMyCell {
    entry  tSig  eEntry;
    call   tSig  cCall;
    attr {
        int32_t count = 0;
    };
    var {
        int32_t state;
    };
};
```
