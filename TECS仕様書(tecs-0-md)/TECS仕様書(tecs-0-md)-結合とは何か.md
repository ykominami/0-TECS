# TECS仕様書(tecs-0-md)-結合とは何か

結合は、**セルの呼び口と受け口を結びつけること**である。結合は**呼び元のセルで指定**し、**コンパイル時に静的に確定**する。同じシグニチャに対応付けられた呼び口と受け口のみが結合できる。

---

## 基本規則

- 結合できるのは同一シグニチャの呼び口と受け口のみ
- 結合は呼び元セル（呼び口を持つ側）で指定する
- すべての結合は静的（コンパイル時確定）
- 実装上は呼び元セルのデータとして受け先へのポインタが埋め込まれる

**なぜ呼び元で指定するか？** ソフトウェア部品では、呼び元から呼び先へのポインタ情報を呼び側のセルに埋め込む形で実現されるため。ハードウェア回路の配線とは異なり、「誰が呼ぶか」を記述側が管理する。

## 結合の形式

| 形式 | 説明 |
|------|------|
| **通常結合** | 呼び口を受け口に直接結合 |
| **オプショナル呼び口** | `optional` 指定された呼び口は未結合のままにできる |
| **固定結合** | CDLで結合先を静的に固定（`=` 記法） |
| **逆結合** | 複合セルタイプ内の逆方向の結合 |
| **接続** (connection) | スループラグイン（RPC等）によりセルが挿入される場合の結合。通常の結合と区別 |

## CDLでの記述

```
cell tA cellA {
    cMyCall = cellB.eMyEntry;   // 呼び口 cMyCall を cellB の受け口 eMyEntry に結合
};
```

## 合流・分流

- **合流**: 複数の呼び口から同一の受け口への結合 → **可能**
- **分流**: 1つの呼び口から複数の受け口 → **不可**（呼び口配列で代替）

シグニチャが形式的に一致しても、関数の意味仕様が一致するかどうかはTECSでは保証しない（定義時に明確にしておく必要がある）。
